1) Formal Setup

Given an image set {x_i}, the miner outputs a scalar ranking score s_i in [0, 100] and interpretable diagnostics. The optimization target is not semantic segmentation quality; it is monotonic ordering by texture suitability for RWTD-like two-to-four region texture transitions. Practically, the miner computes staged signals f_i^A, f_i^geom, f_i^B, f_i^D and fuses them into final score s_i.

2) End-to-End Staged Pipeline

Stage A: Proposal Statistics Gate

Reads annotation masks per image and computes region-size statistics and entropy. Produces stageA_rwtd_score and pass/fail gate.

Code: stages/stage_a_maskstats.py

Geometry Enrichment

Builds geometry proxies that approximate texture suitability: boundary strength, texture occupancy, object occupancy, ambiguity, and region-level structure.

Code: utils/rwtd_reference.py

Stage B: CLIP Retrieval Filter

Computes image-text similarity against positive and negative query banks. Keeps high-score rows and top-k-by-query rows to preserve diversity.

Code: stages/stage_b_clip.py

Stage D: VLM Rubric Audit (Optional)

Applies natural-language rubric plus hard rules to suppress semantic-object dominant false positives and weak-boundary overlays.

Code: stages/stage_d_vlm.py

Final ranking uses multimodal fusion where geometry remains dominant, CLIP is secondary, and VLM acts as a corrective signal with explicit penalties.

Code: cli.py::_apply_multimodal_fusion_score

3) Stage A Equations and Gate Logic

For each image, let mask areas normalized by image area be r_1 ... r_n. Define:

n_score       = min(1, n_masks / 120)
largest_score = 1 - min(1, largest_ratio / 0.65)
median_score  = 1 - min(1, median_ratio / 0.03)
small_score   = min(1, small_frac / 0.55)
entropy_score = min(1, entropy / 4.0)

stageA_rwtd_score = 100 * (
    0.28 * n_score +
    0.24 * largest_score +
    0.18 * median_score +
    0.18 * small_score +
    0.12 * entropy_score
)

Default hard gate:

n_masks >= 80
largest_ratio <= 0.60
median_ratio <= 0.02
small_frac >= 0.30
entropy >= 2.0 (if entropy gate enabled)

Fallback policy allows unknown Stage A rows to pass to Stage B if configured, reducing accidental early-drop caused by parse failures.

4) Geometry Priority Score (RWTD-Calibrated Baseline)

The geometry baseline is explicit and heavily weighted toward texture-boundary evidence:

tex_bonus     = 20 * clip((tex - 0.65) / 0.35, 0, 1)
region_bonus  = +6 if 2<=large_regions<=4, +2 if >4, else -8
strong_bonus  = +3 if strong_boundaries>=1 else -4
obj_penalty   = 180 * clip(obj_frac - 0.002, 0, 1)
amb_penalty   = 120 * clip(ambiguous_frac, 0, 1)
clutter_pen   = 4 * clip(large_regions - 5, 0, +inf)

score_geom = clip(
    0.20 * stageA_rwtd_score +
    0.60 * geom_texture_boundary_score +
    tex_bonus + region_bonus + strong_bonus
    - obj_penalty - amb_penalty - clutter_pen,
0, 100)

This is the "do not overfit to semantics" anchor in the full system.

5) Stage B CLIP Rule (Filtering + Diversity Retention)

Stage B only evaluates candidates that pass upstream gates and minimum resolution (short_side >= 256 by default).

pos = max_q sim(image_emb, pos_query_emb[q])
neg = max_q sim(image_emb, neg_query_emb[q])
clip_score = pos - alpha * neg     (alpha = 0.7 default)

Keep rule is deliberately a union:

pass = (clip_score >= threshold) OR (in top_k of any positive query)
threshold = 0.20
top_k_per_positive_query = 2000

This avoids losing uncommon but valid texture scenes that might be under-scored by one global threshold.

6) Stage D VLM Rubric and Hard Overrides

Stage D scores only a shortlist (top_n from Stage B, default 2000) and returns structured output:

{
  score_0_100: int,
  decision: "match" | "borderline" | "not_match",
  main_reason: string,
  flags: [ ... ]
}

Hard rules then enforce conservative behavior:

  • salient_semantic_subject can force not_match and cap score.
  • Overlay boundary audit can force not_match when boundary structure is weak/noisy.
  • Overlay failure applies cap and penalty; missing overlay can also trigger conservative downgrade.

This stage is used as calibration, not as primary scorer.

7) Multimodal Fusion and Final Status Assignment

clip100 = percentile_robust_scale(stageB_clip_score)

if has_clip:
    fused = 0.92 * base_geom + 0.08 * clip100
if has_vlm:
    fused = 0.90 * base_geom + 0.08 * clip100 + 0.02 * vlm_score

fused -= penalties_from_flags
fused += 2 if decision == "match"
fused -= 2 if decision == "not_match"
fused = clip(fused, 0, 100)

Status labels in this site are derived from final score: selected >= 80, borderline >= 70, otherwise rejected. For operational retrieval, the recommended broader cutoff is currently score >= 60.

8) Deterministic Review Subset Policy (Why 1500 Here)

The ADE20K corpus is larger than what is convenient for live browser review. The command-line run uses --review_limit 1500 and then builds a deterministic balanced subset. Allocation defaults to roughly:

  • 35% selected
  • 25% borderline
  • 40% rejected (mixed high-score hard negatives and diverse lower-score negatives)

The full manifest can still be generated offline; this hosted split is for practical, auditable human inspection.

9) Pseudocode (Inference Path)

for image in dataset:
    A = stage_a_mask_stats(image, annotations)
    G = geometry_features(image, annotations)
    base = texture_priority_score(A, G)

    if stage_b_enabled:
        B = clip_filter(image)
    else:
        B = pass_from_stage_a

    if stage_d_enabled and B.pass and in_stage_b_top_n:
        D = vlm_rubric_and_overlay(image, overlay)
        D = apply_hard_decision_rules(D)
    else:
        D = none

    score = multimodal_fusion(base, B, D)
    status = threshold(score)
    emit(image_id, score, status, diagnostics)

10) Complexity and Compute Profile

Component Dominant Cost Implementation Notes
Stage A Mask-area parsing per image CPU parallelized with ProcessPoolExecutor.
Geometry stage Mask decode + boundary extraction Also produces mask/overlay assets for diagnostics.
Stage B Image embedding and text similarity Embeddings are memmapped to avoid high RAM spikes.
Stage D VLM calls on shortlist only Top-N bound limits latency and API/GPU cost.
